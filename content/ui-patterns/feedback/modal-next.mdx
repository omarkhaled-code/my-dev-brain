## Concept explanation

A modal in Next.js is a `"use client"` component that uses `useState` to control visibility, `useEffect` to handle keyboard events and body scroll lock, and React portals (`createPortal`) to render at the `<body>` level — avoiding z-index and overflow clipping issues. Tailwind CSS handles the backdrop, panel styling, and transitions.

<Callout type="note">
  **Next.js vs React difference:** This component works identically in plain React (Vite). No Next.js-specific APIs are used — `createPortal`, `useState`, and `useEffect` are all standard React. Just copy it into any React project and it works.
</Callout>

## Why it's used

- Confirms destructive actions (delete, logout) without navigating away
- Displays forms (create user, upload file) without leaving the current page
- Shows image previews or detail views inline
- Collects quick input without a full page redirect

## Real-world use cases

- "Are you sure you want to delete this?" confirmation dialog
- "Create new project" form inside a modal
- Image lightbox / preview
- Cookie consent dialog
- Share / invite team member dialog

## Code example

### Basic modal

```tsx
// components/AppModal.tsx
"use client";

import { useEffect, useRef } from "react";
import { createPortal } from "react-dom";
import { X } from "lucide-react";

type Props = {
  isOpen: boolean;
  onClose: () => void;
  title?: string;
  children: React.ReactNode;
  footer?: React.ReactNode;
};

export function AppModal({ isOpen, onClose, title, children, footer }: Props) {

  // Close on ESC key
  useEffect(() => {
    const handleKeydown = (e: KeyboardEvent) => {
      if (e.key === "Escape") onClose();
    };
    if (isOpen) document.addEventListener("keydown", handleKeydown);
    return () => document.removeEventListener("keydown", handleKeydown);
  }, [isOpen, onClose]);

  // Lock body scroll when open
  useEffect(() => {
    document.body.style.overflow = isOpen ? "hidden" : "";
    return () => { document.body.style.overflow = ""; };
  }, [isOpen]);

  if (!isOpen) return null;

  return createPortal(
    <div
      className="fixed inset-0 z-50 flex items-center justify-center p-4"
      role="dialog"
      aria-modal="true"
      aria-label={title}
    >
      {/* Backdrop */}
      <div
        className="absolute inset-0 bg-black/50 backdrop-blur-sm"
        onClick={onClose}
      />

      {/* Modal panel */}
      <div className="relative bg-white dark:bg-neutral-900 rounded-2xl shadow-xl w-full max-w-md z-10 overflow-hidden animate-in fade-in zoom-in-95 duration-200">

        {/* Header */}
        <div className="flex items-center justify-between px-6 py-4 border-b border-neutral-200 dark:border-neutral-800">
          <h2 className="text-base font-semibold text-neutral-900 dark:text-neutral-100">
            {title}
          </h2>
          <button
            onClick={onClose}
            className="p-1.5 rounded-lg text-neutral-400 hover:text-neutral-600 hover:bg-neutral-100 dark:hover:bg-neutral-800 transition-colors"
            aria-label="Close modal"
          >
            <X size={16} />
          </button>
        </div>

        {/* Body */}
        <div className="px-6 py-5">
          {children}
        </div>

        {/* Footer */}
        {footer && (
          <div className="px-6 py-4 border-t border-neutral-200 dark:border-neutral-800 flex items-center justify-end gap-3">
            {footer}
          </div>
        )}
      </div>
    </div>,
    document.body
  );
}
```

### Confirmation modal (delete dialog)

```tsx
// components/ConfirmModal.tsx
"use client";

import { useEffect } from "react";
import { createPortal } from "react-dom";

type Props = {
  isOpen: boolean;
  title: string;
  message: string;
  confirmLabel?: string;
  danger?: boolean;
  onConfirm: () => void;
  onCancel: () => void;
};

export function ConfirmModal({
  isOpen,
  title,
  message,
  confirmLabel = "Confirm",
  danger = false,
  onConfirm,
  onCancel,
}: Props) {

  useEffect(() => {
    const handleKeydown = (e: KeyboardEvent) => {
      if (e.key === "Escape") onCancel();
    };
    if (isOpen) document.addEventListener("keydown", handleKeydown);
    return () => document.removeEventListener("keydown", handleKeydown);
  }, [isOpen, onCancel]);

  useEffect(() => {
    document.body.style.overflow = isOpen ? "hidden" : "";
    return () => { document.body.style.overflow = ""; };
  }, [isOpen]);

  if (!isOpen) return null;

  return createPortal(
    <div
      className="fixed inset-0 z-50 flex items-center justify-center p-4"
      role="dialog"
      aria-modal="true"
    >
      {/* Backdrop */}
      <div
        className="absolute inset-0 bg-black/50 backdrop-blur-sm"
        onClick={onCancel}
      />

      {/* Panel */}
      <div className="relative bg-white dark:bg-neutral-900 rounded-2xl shadow-xl w-full max-w-sm z-10 p-6 animate-in fade-in zoom-in-95 duration-200">

        {/* Icon */}
        <div className={`w-12 h-12 rounded-full flex items-center justify-center mx-auto mb-4 ${danger ? "bg-red-100" : "bg-amber-100"}`}>
          <svg
            className={`w-6 h-6 ${danger ? "text-red-600" : "text-amber-600"}`}
            fill="none" stroke="currentColor" viewBox="0 0 24 24"
          >
            <path strokeLinecap="round" strokeLinejoin="round" strokeWidth={2}
              d="M12 9v2m0 4h.01m-6.938 4h13.856c1.54 0 2.502-1.667 1.732-3L13.732 4c-.77-1.333-2.694-1.333-3.464 0L3.34 16c-.77 1.333.192 3 1.732 3z"
            />
          </svg>
        </div>

        <h3 className="text-base font-semibold text-neutral-900 dark:text-neutral-100 text-center mb-2">
          {title}
        </h3>
        <p className="text-sm text-neutral-500 text-center mb-6">
          {message}
        </p>

        <div className="flex gap-3">
          <button
            onClick={onCancel}
            className="flex-1 px-4 py-2 text-sm font-medium text-neutral-600 border border-neutral-200 rounded-lg hover:bg-neutral-50 transition-colors"
          >
            Cancel
          </button>
          <button
            onClick={onConfirm}
            className={`flex-1 px-4 py-2 text-sm font-medium text-white rounded-lg transition-colors ${danger ? "bg-red-600 hover:bg-red-700" : "bg-blue-600 hover:bg-blue-700"}`}
          >
            {confirmLabel}
          </button>
        </div>
      </div>
    </div>,
    document.body
  );
}
```

## Common mistakes

<WrongRight>
<Wrong>
```tsx
// Not using createPortal — modal renders inside a parent
// with overflow:hidden and gets clipped, z-index fights everywhere
export function AppModal({ isOpen, children }) {
  if (!isOpen) return null;
  return (
    <div className="fixed inset-0 z-50">
      {/* clipped by parent overflow:hidden ❌ */}
      {children}
    </div>
  );
}
```
</Wrong>
<Right>
```tsx
// Always use createPortal to render at body level
// No more z-index or overflow issues ever
import { createPortal } from "react-dom";

export function AppModal({ isOpen, children }) {
  if (!isOpen) return null;
  return createPortal(
    <div className="fixed inset-0 z-50">
      {children} {/* renders directly on body ✅ */}
    </div>,
    document.body
  );
}
```
</Right>
</WrongRight>

<WrongRight>
<Wrong>
```tsx
// Forgetting to clean up event listeners — causes memory leaks
// and the handler fires even after modal unmounts
useEffect(() => {
  document.addEventListener("keydown", handleKeydown); // ❌ no cleanup
}, [isOpen]);
```
</Wrong>
<Right>
```tsx
// Always return a cleanup function from useEffect
useEffect(() => {
  if (isOpen) document.addEventListener("keydown", handleKeydown);
  return () => document.removeEventListener("keydown", handleKeydown); // ✅
}, [isOpen]);
```
</Right>
</WrongRight>

## Best practices

- Always use `createPortal(…, document.body)` to avoid z-index and overflow clipping
- Lock body scroll with `document.body.style.overflow = "hidden"` — restore in cleanup
- Always clean up `addEventListener` in the `useEffect` return function
- Add `role="dialog"` and `aria-modal="true"` for accessibility
- Clicking the backdrop should close the modal — clicking inside the panel should not
- Pass `footer` as a prop rather than hardcoding buttons — keeps the modal reusable
- Reset form state when modal closes using a `useEffect` watching `isOpen`

<Callout type="tip">
  To reset a form when the modal closes, watch `isOpen` in a `useEffect`:
```tsx
  useEffect(() => {
    if (!isOpen) setFormData({ name: "", email: "" });
  }, [isOpen]);
```
</Callout>

## Interview questions

<InterviewQ question="What is React's createPortal and why use it for modals?" difficulty="mid">
  `createPortal` renders a component's output into a different DOM node than its parent — usually `document.body`. For modals this is critical because `fixed` positioning and `z-index` are affected by the stacking context of parent elements. If the modal renders inside a parent with `overflow: hidden` or `transform`, it gets clipped or has z-index conflicts. Portals escape the parent's stacking context entirely.
</InterviewQ>

<InterviewQ question="Why must you clean up event listeners in useEffect?" difficulty="easy">
  If you add an event listener in `useEffect` without removing it when the component unmounts, the handler keeps firing even after the component is gone — causing memory leaks and potential errors when the handler tries to update state on an unmounted component. Always return a cleanup function that calls `removeEventListener` with the exact same function reference.
</InterviewQ>

<InterviewQ question="What is the difference between createPortal in React and Teleport in Vue?" difficulty="mid">
  They solve the same problem — rendering content outside the component's DOM parent. React's `createPortal(children, container)` is a function you call inside `render`/`return`, passing the target DOM node directly. Vue's `<Teleport to="body">` is a built-in component wrapping the content declaratively in the template. Both render the content at `body` level while keeping the component logically in its parent tree for event bubbling and context.
</InterviewQ>

## My notes

<MyNotes>
  Two things I always forget in React modals:

  1. `createPortal` — without it I get mysterious clipping bugs inside layouts that have `overflow: hidden`. Always portal to `document.body`.

  2. Cleanup in `useEffect` — I've had handlers firing after unmount causing "Can't perform state update on unmounted component" warnings. Always return the cleanup function.

  For the animation — Tailwind's `animate-in fade-in zoom-in-95` requires the `tailwindcss-animate` plugin. Install it with `npm install tailwindcss-animate` and add `@plugin "tailwindcss-animate"` to your `globals.css`. If you don't want the plugin, just remove the animation classes — the modal still works perfectly.
</MyNotes>
