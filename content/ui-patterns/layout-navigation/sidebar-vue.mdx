## Concept explanation

A sidebar is a vertical navigation panel — usually fixed on the left side of the screen. On desktop it stays visible. On mobile it slides in as an overlay when triggered. In Vue 3 we manage the open/collapsed state with `ref()`, use `useRoute()` to highlight the active link, and Tailwind handles the responsive behavior and transitions.

<Callout type="note">
  **Vue vs Nuxt difference:** In Nuxt, `useRoute()` and `navigateTo()` are auto-imported — no import needed. Use `NuxtLink` instead of `RouterLink`. Everything else is identical.
</Callout>

## Why it's used

- Better for apps with many navigation items that don't fit in a top navbar
- Keeps the main content area full-width and uncluttered
- Collapsible sidebar saves screen space on smaller monitors
- Standard pattern for dashboards and admin panels

## Real-world use cases

- Admin dashboards (users, orders, settings, reports)
- Documentation sites with many sections
- SaaS apps with multiple modules
- Project management tools like Notion or Linear

## Code example

### Basic sidebar

```vue
<!-- components/AppSidebar.vue -->
<script setup lang="ts">
import { ref } from 'vue'
import { useRoute } from 'vue-router'
import {
  HomeIcon,
  UsersIcon,
  FolderIcon,
  ChartBarIcon,
  CogIcon,
} from '@heroicons/vue/24/outline'

const route = useRoute()
const isOpen = ref(true)

const links = [
  { label: 'Dashboard', href: '/',        icon: HomeIcon },
  { label: 'Users',     href: '/users',   icon: UsersIcon },
  { label: 'Projects',  href: '/projects',icon: FolderIcon },
  { label: 'Analytics', href: '/analytics',icon: ChartBarIcon },
  { label: 'Settings',  href: '/settings',icon: CogIcon },
]

const isActive = (href: string) => route.path === href
</script>

<template>
  <div class="flex min-h-screen">

    <!-- Mobile overlay -->
    <div
      v-if="isOpen"
      class="fixed inset-0 bg-black/50 z-20 lg:hidden"
      @click="isOpen = false"
    />

    <!-- Sidebar -->
    <aside
      :class="[
        'fixed top-0 left-0 h-full z-30 bg-white border-r border-neutral-200',
        'transition-transform duration-300 ease-in-out',
        'w-64 flex flex-col',
        isOpen ? 'translate-x-0' : '-translate-x-full',
        'lg:translate-x-0 lg:static lg:z-auto'
      ]"
    >
      <!-- Logo -->
      <div class="px-5 py-5 border-b border-neutral-200">
        <RouterLink to="/" class="flex items-center gap-2">
          <span class="text-lg font-bold">🧠 DevBrain</span>
        </RouterLink>
      </div>

      <!-- Navigation -->
      <nav class="flex-1 px-3 py-4 space-y-0.5 overflow-y-auto">
        <RouterLink
          v-for="link in links"
          :key="link.href"
          :to="link.href"
          @click="isOpen = false"
          :class="[
            'flex items-center gap-3 px-3 py-2 rounded-lg text-sm font-medium transition-colors',
            isActive(link.href)
              ? 'bg-blue-50 text-blue-700'
              : 'text-neutral-600 hover:bg-neutral-100 hover:text-neutral-900'
          ]"
        >
          <component :is="link.icon" class="w-5 h-5 shrink-0" />
          {{ link.label }}
        </RouterLink>
      </nav>

      <!-- Bottom user section -->
      <div class="px-3 py-4 border-t border-neutral-200">
        <div class="flex items-center gap-3 px-3 py-2 rounded-lg hover:bg-neutral-100 cursor-pointer">
          <div class="w-8 h-8 rounded-full bg-blue-600 flex items-center justify-center text-white text-sm font-medium shrink-0">
            O
          </div>
          <div class="flex-1 min-w-0">
            <p class="text-sm font-medium text-neutral-900 truncate">Omar</p>
            <p class="text-xs text-neutral-400 truncate">omar@example.com</p>
          </div>
        </div>
      </div>
    </aside>

    <!-- Main content -->
    <div class="flex-1 min-w-0">

      <!-- Mobile topbar with hamburger -->
      <header class="lg:hidden flex items-center gap-4 px-4 h-14 border-b border-neutral-200 bg-white sticky top-0 z-10">
        <button
          @click="isOpen = true"
          class="p-2 rounded-lg hover:bg-neutral-100 transition-colors"
          aria-label="Open sidebar"
        >
          <svg class="w-5 h-5" fill="none" stroke="currentColor" viewBox="0 0 24 24">
            <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M4 6h16M4 12h16M4 18h16"/>
          </svg>
        </button>
        <span class="text-lg font-bold">🧠 DevBrain</span>
      </header>

      <!-- Page content -->
      <main class="p-6">
        <slot />
      </main>
    </div>

  </div>
</template>
```

### With collapsible icon-only mode

```vue
<!-- components/AppSidebar.vue (collapsible) -->
<script setup lang="ts">
import { ref } from 'vue'
import { useRoute } from 'vue-router'

const route = useRoute()
const collapsed = ref(false)
const mobileOpen = ref(false)

const links = [
  { label: 'Dashboard', href: '/',         emoji: '🏠' },
  { label: 'Users',     href: '/users',    emoji: '👥' },
  { label: 'Projects',  href: '/projects', emoji: '📁' },
  { label: 'Analytics', href: '/analytics',emoji: '📊' },
  { label: 'Settings',  href: '/settings', emoji: '⚙️' },
]

const isActive = (href: string) => route.path === href
</script>

<template>
  <div class="flex min-h-screen">

    <!-- Mobile overlay -->
    <div
      v-if="mobileOpen"
      class="fixed inset-0 bg-black/50 z-20 lg:hidden"
      @click="mobileOpen = false"
    />

    <!-- Sidebar -->
    <aside
      :class="[
        'fixed top-0 left-0 h-full z-30 bg-white border-r border-neutral-200 flex flex-col',
        'transition-all duration-300 ease-in-out',
        collapsed ? 'w-16' : 'w-64',
        mobileOpen ? 'translate-x-0' : '-translate-x-full',
        'lg:translate-x-0 lg:static lg:z-auto'
      ]"
    >
      <!-- Logo + collapse toggle -->
      <div class="flex items-center justify-between px-3 py-4 border-b border-neutral-200 h-16">
        <span v-if="!collapsed" class="text-lg font-bold truncate">🧠 DevBrain</span>
        <button
          @click="collapsed = !collapsed"
          class="hidden lg:flex p-1.5 rounded-lg hover:bg-neutral-100 transition-colors ml-auto"
          :aria-label="collapsed ? 'Expand sidebar' : 'Collapse sidebar'"
        >
          <svg
            class="w-4 h-4 text-neutral-500 transition-transform duration-300"
            :class="collapsed ? 'rotate-180' : ''"
            fill="none" stroke="currentColor" viewBox="0 0 24 24"
          >
            <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M15 19l-7-7 7-7"/>
          </svg>
        </button>
      </div>

      <!-- Nav links -->
      <nav class="flex-1 px-2 py-4 space-y-0.5 overflow-y-auto">
        <RouterLink
          v-for="link in links"
          :key="link.href"
          :to="link.href"
          @click="mobileOpen = false"
          :class="[
            'flex items-center gap-3 px-2 py-2 rounded-lg text-sm font-medium transition-colors',
            collapsed ? 'justify-center' : '',
            isActive(link.href)
              ? 'bg-blue-50 text-blue-700'
              : 'text-neutral-600 hover:bg-neutral-100 hover:text-neutral-900'
          ]"
          :title="collapsed ? link.label : ''"
        >
          <span class="text-lg shrink-0">{{ link.emoji }}</span>
          <span v-if="!collapsed" class="truncate">{{ link.label }}</span>
        </RouterLink>
      </nav>
    </aside>

    <!-- Main content -->
    <div class="flex-1 min-w-0 flex flex-col">
      <header class="lg:hidden flex items-center gap-4 px-4 h-14 border-b border-neutral-200 bg-white sticky top-0 z-10">
        <button
          @click="mobileOpen = true"
          class="p-2 rounded-lg hover:bg-neutral-100"
          aria-label="Open sidebar"
        >
          <svg class="w-5 h-5" fill="none" stroke="currentColor" viewBox="0 0 24 24">
            <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M4 6h16M4 12h16M4 18h16"/>
          </svg>
        </button>
        <span class="font-bold">🧠 DevBrain</span>
      </header>

      <main class="flex-1 p-6">
        <slot />
      </main>
    </div>

  </div>
</template>
```

### With nested menu groups

```vue
<!-- components/AppSidebar.vue (nested groups) -->
<script setup lang="ts">
import { ref } from 'vue'
import { useRoute } from 'vue-router'

const route = useRoute()
const openGroups = ref<string[]>(['Frontend'])

const navigation = [
  {
    group: 'Frontend',
    items: [
      { label: 'React',      href: '/frontend/react' },
      { label: 'Vue',        href: '/frontend/vue' },
      { label: 'TypeScript', href: '/frontend/typescript' },
    ],
  },
  {
    group: 'Backend',
    items: [
      { label: 'Laravel',   href: '/backend/laravel' },
      { label: 'Database',  href: '/backend/database' },
    ],
  },
]

const toggleGroup = (group: string) => {
  openGroups.value = openGroups.value.includes(group)
    ? openGroups.value.filter(g => g !== group)
    : [...openGroups.value, group]
}

const isActive = (href: string) => route.path.startsWith(href)
</script>

<template>
  <aside class="w-64 h-screen sticky top-0 bg-white border-r border-neutral-200 flex flex-col overflow-y-auto">
    <div class="px-5 py-5 border-b border-neutral-200">
      <span class="text-lg font-bold">🧠 DevBrain</span>
    </div>

    <nav class="flex-1 px-3 py-4">
      <div v-for="section in navigation" :key="section.group" class="mb-2">

        <!-- Group toggle -->
        <button
          @click="toggleGroup(section.group)"
          class="w-full flex items-center justify-between px-2 py-1.5 text-xs font-semibold text-neutral-500 uppercase tracking-wider hover:text-neutral-900 transition-colors"
        >
          {{ section.group }}
          <svg
            class="w-3.5 h-3.5 transition-transform duration-200"
            :class="openGroups.includes(section.group) ? 'rotate-180' : ''"
            fill="none" stroke="currentColor" viewBox="0 0 24 24"
          >
            <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"/>
          </svg>
        </button>

        <!-- Group items -->
        <div v-if="openGroups.includes(section.group)" class="mt-1 ml-2 pl-3 border-l border-neutral-200 space-y-0.5">
          <RouterLink
            v-for="item in section.items"
            :key="item.href"
            :to="item.href"
            :class="[
              'block px-2 py-1.5 rounded-lg text-sm transition-colors',
              isActive(item.href)
                ? 'bg-blue-50 text-blue-700 font-medium'
                : 'text-neutral-500 hover:text-neutral-900 hover:bg-neutral-100'
            ]"
          >
            {{ item.label }}
          </RouterLink>
        </div>

      </div>
    </nav>
  </aside>
</template>
```

## Common mistakes

<WrongRight>
<Wrong>
```vue
<!-- No mobile overlay — sidebar opens but user can't close it
     by clicking outside, terrible UX on mobile -->
<aside v-if="isOpen" class="fixed left-0 top-0 h-full w-64 bg-white">
  ...
</aside>
```
</Wrong>
<Right>
```vue
<!-- Always add a dark overlay behind the sidebar on mobile
     so user can tap outside to close it -->
<div
  v-if="isOpen"
  class="fixed inset-0 bg-black/50 z-20 lg:hidden"
  @click="isOpen = false"
/>
<aside class="fixed left-0 top-0 h-full w-64 bg-white z-30">
  ...
</aside>
```
</Right>
</WrongRight>

<WrongRight>
<Wrong>
```vue
<!-- Forgetting to close sidebar when a link is clicked on mobile
     sidebar stays open after navigation -->
<RouterLink to="/users">Users</RouterLink>
```
</Wrong>
<Right>
```vue
<!-- Always close mobile sidebar on link click -->
<RouterLink
  to="/users"
  @click="isOpen = false"
>
  Users
</RouterLink>
```
</Right>
</WrongRight>

## Best practices

- Always use `z-30` on the sidebar and `z-20` on the overlay so they stack correctly
- Use `lg:translate-x-0 lg:static` so the sidebar is always visible on desktop without JS
- Add `overflow-y-auto` on the nav so it scrolls independently if you have many links
- Use `transition-transform duration-300` for smooth slide-in animation on mobile
- Add `title` attribute on collapsed icon-only links so users see a tooltip on hover
- Store collapsed state in `localStorage` so it persists across page refreshes

<Callout type="tip">
  To persist the collapsed state across sessions add this to your setup:
```ts
  const collapsed = ref(localStorage.getItem('sidebar-collapsed') === 'true')
  watch(collapsed, (val) => localStorage.setItem('sidebar-collapsed', String(val)))
```
</Callout>

## Interview questions

<InterviewQ question="How do you handle sidebar state across multiple pages in Vue?" difficulty="mid">
  Use Pinia to store the sidebar open/collapsed state globally. This way any component can toggle it and the state persists across route changes without prop drilling. For simple cases a composable with a module-level `ref` also works.
</InterviewQ>

<InterviewQ question="What is the difference between fixed and sticky positioning for a sidebar?" difficulty="easy">
  `fixed` positions the element relative to the viewport — it stays in place even when the page scrolls, but it's taken out of document flow so you need to add margin to the main content. `sticky` keeps the element in document flow and sticks when it reaches a scroll threshold — better for sidebars that are part of a flex layout.
</InterviewQ>

<InterviewQ question="How do you make a sidebar accessible?" difficulty="mid">
  Add `aria-label="Sidebar navigation"` to the aside element. Add `aria-expanded` to any toggle button. Ensure all links are keyboard focusable. When the mobile sidebar opens, trap focus inside it so keyboard users can't tab to hidden content behind the overlay. Close it with the Escape key by listening to `keydown` on the document.
</InterviewQ>

## My notes

<MyNotes>
  The z-index stacking always trips me up. Remember: overlay is z-20, sidebar is z-30, and any modals on top should be z-50. If something appears behind the sidebar unexpectedly, check the z-index first.

  For Nuxt — use `NuxtLink` instead of `RouterLink` and `useRoute()` is auto-imported so no import needed. The rest is identical.

  If you use Pinia for global sidebar state, create a `useSidebarStore` and keep `isOpen` and `collapsed` there — makes it easy to toggle from anywhere including a mobile topbar that lives in a completely different component tree.
</MyNotes>
