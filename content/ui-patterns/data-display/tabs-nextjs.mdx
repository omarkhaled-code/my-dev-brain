## Concept explanation

Tabs in Next.js can be built two ways — **client-side** with `useState` for simple cases, and **URL-based** with `useSearchParams` for production apps where the active tab should be bookmarkable and shareable. The component accepts a `tabs` array and uses a `renderPanel` render prop pattern so each tab's content stays decoupled from the tabs component itself.

<Callout type="note">
  **Next.js vs React difference:** The `useState` approach works identically in plain React. The URL-based approach uses `useSearchParams` and `useRouter` from `next/navigation` — in plain React replace these with `useSearchParams` and `useNavigate` from `react-router-dom`.
</Callout>

## Why it's used

- Organizes related content into sections without page navigation
- Keeps users in context — no full page reload between sections
- Reduces visual clutter by showing one section at a time
- Standard pattern users know from browsers, IDEs, and dashboards

## Real-world use cases

- User profile page (Overview, Activity, Settings)
- Product page (Description, Reviews, Specifications)
- Dashboard (Analytics, Reports, Exports)
- Settings page (General, Security, Notifications, Billing)
- Code editor (multiple open files)

## Code example

### Basic tabs component

```tsx
// components/Tabs.tsx
"use client";

import { useState, type ReactNode } from "react";

type Tab = {
  value: string;
  label: string;
  icon?: string;
  disabled?: boolean;
};

type Props = {
  tabs: Tab[];
  defaultValue?: string;
  panels: Record<string, ReactNode>;
};

export function Tabs({ tabs, defaultValue, panels }: Props) {
  const [active, setActive] = useState(defaultValue ?? tabs[0]?.value);

  const handleKeyDown = (
    e: React.KeyboardEvent,
    currentIndex: number
  ) => {
    if (e.key === "ArrowLeft") {
      const prev = tabs
        .slice(0, currentIndex)
        .reverse()
        .find((t) => !t.disabled);
      if (prev) setActive(prev.value);
    }
    if (e.key === "ArrowRight") {
      const next = tabs.slice(currentIndex + 1).find((t) => !t.disabled);
      if (next) setActive(next.value);
    }
  };

  return (
    <div>
      {/* Tab list */}
      <div
        className="flex border-b border-neutral-200 dark:border-neutral-800"
        role="tablist"
      >
        {tabs.map((tab, i) => (
          <button
            key={tab.value}
            onClick={() => !tab.disabled && setActive(tab.value)}
            onKeyDown={(e) => handleKeyDown(e, i)}
            disabled={tab.disabled}
            aria-selected={active === tab.value}
            aria-controls={`panel-${tab.value}`}
            role="tab"
            className={`
              flex items-center gap-2 px-4 py-3 text-sm font-medium
              border-b-2 -mb-px transition-colors
              ${active === tab.value
                ? "border-blue-600 text-blue-600 dark:border-blue-400 dark:text-blue-400"
                : "border-transparent text-neutral-500 dark:text-neutral-400 hover:text-neutral-900 dark:hover:text-neutral-100 hover:border-neutral-300 dark:hover:border-neutral-600"
              }
              ${tab.disabled ? "opacity-40 cursor-not-allowed" : "cursor-pointer"}
            `}
          >
            {tab.icon && <span>{tab.icon}</span>}
            {tab.label}
          </button>
        ))}
      </div>

      {/* Tab panels */}
      <div className="mt-4">
        {tabs.map((tab) => (
          <div
            key={tab.value}
            id={`panel-${tab.value}`}
            role="tabpanel"
            hidden={active !== tab.value}
          >
            {panels[tab.value]}
          </div>
        ))}
      </div>
    </div>
  );
}
```

### URL-based tabs (production approach)

```tsx
// components/TabsURL.tsx
"use client";

import { useRouter, usePathname, useSearchParams } from "next/navigation";
import { type ReactNode, useCallback } from "react";

type Tab = {
  value: string;
  label: string;
  icon?: string;
};

type Props = {
  tabs: Tab[];
  panels: Record<string, ReactNode>;
  paramName?: string;
};

export function TabsURL({ tabs, panels, paramName = "tab" }: Props) {
  const router = useRouter();
  const pathname = usePathname();
  const searchParams = useSearchParams();

  const active = searchParams.get(paramName) ?? tabs[0]?.value;

  const setTab = useCallback(
    (value: string) => {
      const params = new URLSearchParams(searchParams.toString());
      params.set(paramName, value);
      router.push(`${pathname}?${params.toString()}`);
    },
    [pathname, router, searchParams, paramName]
  );

  return (
    <div>
      {/* Tab list */}
      <div
        className="flex border-b border-neutral-200 dark:border-neutral-800"
        role="tablist"
      >
        {tabs.map((tab) => (
          <button
            key={tab.value}
            onClick={() => setTab(tab.value)}
            aria-selected={active === tab.value}
            role="tab"
            className={`
              flex items-center gap-2 px-4 py-3 text-sm font-medium
              border-b-2 -mb-px transition-colors
              ${active === tab.value
                ? "border-blue-600 text-blue-600 dark:border-blue-400 dark:text-blue-400"
                : "border-transparent text-neutral-500 dark:text-neutral-400 hover:text-neutral-900 dark:hover:text-neutral-100 hover:border-neutral-300"
              }
            `}
          >
            {tab.icon && <span>{tab.icon}</span>}
            {tab.label}
          </button>
        ))}
      </div>

      {/* Panels */}
      <div className="mt-6">
        {tabs.map((tab) => (
          <div key={tab.value} hidden={active !== tab.value}>
            {panels[tab.value]}
          </div>
        ))}
      </div>
    </div>
  );
}
```

### How to use URL-based tabs

```tsx
// app/settings/page.tsx
import { Suspense } from "react";
import { TabsURL } from "@/components/TabsURL";

const tabs = [

  return (
    <div className="max-w-3xl">
      <h1 className="text-2xl font-bold mb-6">Settings</h1>

            general: (

              </div>

              <div>
                <h2 className="text-lg font-semibold mb-4">Notifications</h2>

              <div>

              </div>
            ),

        />
      </Suspense>
    </div>
  );
}
```

## Common mistakes

<WrongRight>
<Wrong>
```tsx
// Conditionally rendering panels with && operator
// Destroys and remounts the panel on every tab switch
// loses form state, scroll position, fetch cache
{active === "settings" && (
  <SettingsForm /> // ❌ remounts every time
)}
```
</Wrong>
<Right>
```tsx
// Use the hidden attribute — keeps DOM alive
// preserves form state and component state
<div hidden={active !== "settings"}>
  <SettingsForm /> {/* ✅ stays mounted */}
</div>
```
</Right>
</WrongRight>

<WrongRight>
<Wrong>
```tsx
// Forgetting Suspense around useSearchParams
// Next.js throws a build error without it
export default function SettingsPage() {
  return <TabsURL tabs={tabs} panels={panels} /> // ❌ no Suspense
}
```
</Wrong>
<Right>
```tsx
// Always wrap components using useSearchParams in Suspense
export default function SettingsPage() {
  return (
    <Suspense fallback={null}>
      <TabsURL tabs={tabs} panels={panels} /> {/* ✅ */}
    </Suspense>
  )
}
```
</Right>
</WrongRight>

## Best practices

- Use `hidden` attribute instead of conditional rendering — preserves DOM state between tab switches
- Use URL-based tabs for settings pages — active tab is bookmarkable and preserved on refresh
- Always wrap `useSearchParams` in a `Suspense` boundary
- Add `role="tablist"`, `role="tab"`, `role="tabpanel"` and `aria-selected` for accessibility
- Support arrow key navigation for keyboard users
- Support `disabled` tabs with visual feedback
- Use `paramName` prop in `TabsURL` so multiple tab groups on the same page don't conflict

<Callout type="tip">
  The `hidden` HTML attribute is better than `display: none` via Tailwind for tab panels — it's semantically correct, works with `aria-hidden` automatically, and is supported in all modern browsers. React renders `hidden={true}` as the `hidden` attribute correctly.
</Callout>

## Interview questions

<InterviewQ question="Why use the hidden attribute instead of conditional rendering for tab panels?" difficulty="easy">
  Conditional rendering with `&&` or ternary unmounts the component when false — all state (form inputs, scroll position, fetched data) is lost. When the user switches back, the component remounts from scratch. The `hidden` attribute keeps the DOM element alive but visually hidden — state is preserved. This matters for tabs containing forms, infinite scroll lists, or components that fetch their own data.
</InterviewQ>

<InterviewQ question="Why do you need Suspense around useSearchParams in Next.js?" difficulty="hard">
  `useSearchParams` reads from the URL during rendering, which makes the component dynamic — it can't be statically rendered. Next.js App Router requires components that opt into dynamic rendering to be wrapped in `Suspense` so the static shell of the page can be sent immediately while the dynamic part streams in. Without `Suspense`, Next.js throws a build error because it can't statically render the page.
</InterviewQ>

<InterviewQ question="How would you handle two separate tab groups on the same page?" difficulty="mid">
  Use different `paramName` values for each tab group in `TabsURL` — for example `paramName="profileTab"` and `paramName="settingsTab"`. Each group reads and writes its own URL param independently. The URL becomes `?profileTab=activity&settingsTab=security` — both tabs are bookmarkable and independent.
</InterviewQ>

## My notes

<MyNotes>
  The `hidden` attribute approach is something I only learned after breaking a form inside tabs. User filled in their name and email, switched to the Security tab, came back, and the form was empty. Switched from `&&` to `hidden` and it works perfectly.

  The `Suspense` wrapper for `useSearchParams` is easy to forget — Next.js gives a cryptic error at build time without it. Just always wrap any component using `useSearchParams` in `Suspense`.

  **Vue vs React difference:** Vue uses `v-show` to keep DOM alive. React uses the `hidden` attribute. Same concept — both keep the element in the DOM but hide it visually. Vue also has `v-if` vs `v-show` as a familiar distinction. In React you choose between `&&`/ternary (unmounts) vs `hidden` (keeps alive).
</MyNotes>
