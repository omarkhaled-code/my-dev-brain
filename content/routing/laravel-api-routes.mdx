## Concept explanation

Laravel provides `Route::resource()` and `Route::apiResource()` to generate all standard CRUD routes for a resource in one line. Route model binding automatically resolves Eloquent models from route parameters — no manual `find()` calls needed. API versioning organizes your API under `/api/v1/` and `/api/v2/` so you can evolve the API without breaking existing clients.

## Why it's used

- `Route::apiResource()` generates 5 standard routes in one line — no repetition
- Route model binding removes boilerplate `User::find($id)` from every controller
- Automatic 404 when model not found — no manual `abort(404)` needed
- API versioning lets you ship breaking changes without breaking old clients

## Real-world use cases

- REST API for a mobile app
- Headless CMS API consumed by a frontend
- Multi-version public API
- SaaS backend with versioned endpoints
- Inertia.js app with API endpoints for async operations

## Code example

### Route::resource vs Route::apiResource

```php
// Route::resource — generates 7 routes (includes create and edit form pages)
Route::resource('users', UserController::class);

// Route::apiResource — generates 5 routes (no create/edit — APIs don't need HTML forms)
Route::apiResource('users', UserController::class);
```

```bash
# Routes generated by Route::apiResource('users', UserController::class):
# GET    /api/users          → UserController@index   (users.index)
# POST   /api/users          → UserController@store   (users.store)
# GET    /api/users/{user}   → UserController@show    (users.show)
# PUT    /api/users/{user}   → UserController@update  (users.update)
# DELETE /api/users/{user}   → UserController@destroy (users.destroy)
```

### Controller for apiResource

```php
<?php
// app/Http/Controllers/Api/UserController.php

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Models\User;
use Illuminate\Http\Request;
use Illuminate\Http\JsonResponse;

class UserController extends Controller
{
    // GET /api/users
    public function index(): JsonResponse
    {
        $users = User::paginate(15);
        return response()->json($users);
    }

    // POST /api/users
    public function store(Request $request): JsonResponse
    {
        $validated = $request->validate([
            'name'     => 'required|string|max:255',
            'email'    => 'required|email|unique:users',
            'password' => 'required|min:8',
        ]);

        $user = User::create($validated);
        return response()->json($user, 201);
    }

    // GET /api/users/{user}
    // Route model binding — Laravel auto-injects the User model
    public function show(User $user): JsonResponse
    {
        return response()->json($user);
    }

    // PUT /api/users/{user}
    public function update(Request $request, User $user): JsonResponse
    {
        $validated = $request->validate([
            'name'  => 'sometimes|string|max:255',
            'email' => 'sometimes|email|unique:users,email,' . $user->id,
        ]);

        $user->update($validated);
        return response()->json($user);
    }

    // DELETE /api/users/{user}
    public function destroy(User $user): JsonResponse
    {
        $user->delete();
        return response()->json(null, 204);
    }
}
```

### Route model binding — automatic

```php
// routes/api.php
// Laravel sees {user} and automatically finds User::find($id)
// Returns 404 JSON if not found
Route::apiResource('users', UserController::class);

// In the controller — no find() needed
public function show(User $user): JsonResponse
{
    return response()->json($user); // $user is already resolved ✅
}

// Instead of this boilerplate:
public function show(string $id): JsonResponse
{
    $user = User::find($id);       // ❌ manual lookup
    if (!$user) abort(404);        // ❌ manual 404
    return response()->json($user);
}
```

### Route model binding — custom key

```php
// By default binds by primary key (id)
// You can bind by any column

// Option 1 — in the route definition
Route::get('/posts/{post:slug}', [PostController::class, 'show']);
// Now {post:slug} finds Post where slug = $slug

// Option 2 — in the model
class Post extends Model
{
    // Change the default binding key
    public function getRouteKeyName(): string
    {
        return 'slug';
    }
}
// Now Route::apiResource('posts', PostController::class)
// binds {post} by slug automatically
```

### Nested resource routes

```php
// GET    /api/posts/{post}/comments          → CommentController@index
// POST   /api/posts/{post}/comments          → CommentController@store
// GET    /api/posts/{post}/comments/{comment}→ CommentController@show
// PUT    /api/posts/{post}/comments/{comment}→ CommentController@update
// DELETE /api/posts/{post}/comments/{comment}→ CommentController@destroy

Route::apiResource('posts.comments', CommentController::class);

// Controller
class CommentController extends Controller
{
    public function index(Post $post): JsonResponse
    {
        return response()->json($post->comments()->paginate(15));
    }

    public function store(Request $request, Post $post): JsonResponse
    {
        $comment = $post->comments()->create($request->validated());
        return response()->json($comment, 201);
    }

    public function show(Post $post, Comment $comment): JsonResponse
    {
        return response()->json($comment);
    }
}
```

### Partial resource — only specific routes

```php
// Only generate specific routes
Route::apiResource('users', UserController::class)
    ->only(['index', 'show']);

// Exclude specific routes
Route::apiResource('users', UserController::class)
    ->except(['destroy']);
```

### API versioning

```php
// routes/api.php
use Illuminate\Support\Facades\Route;

// Version 1
Route::prefix('v1')
    ->name('v1.')
    ->group(function () {
        Route::apiResource('users',    \App\Http\Controllers\Api\V1\UserController::class);
        Route::apiResource('posts',    \App\Http\Controllers\Api\V1\PostController::class);
        Route::apiResource('products', \App\Http\Controllers\Api\V1\ProductController::class);
    });

// Version 2 — new version, old still works
Route::prefix('v2')
    ->name('v2.')
    ->group(function () {
        Route::apiResource('users',    \App\Http\Controllers\Api\V2\UserController::class);
        Route::apiResource('posts',    \App\Http\Controllers\Api\V2\PostController::class);
        Route::apiResource('products', \App\Http\Controllers\Api\V2\ProductController::class);
    });
```

```
app/Http/Controllers/Api/
├── V1/
│   ├── UserController.php
│   ├── PostController.php
│   └── ProductController.php
└── V2/
    ├── UserController.php    ← updated response shape
    ├── PostController.php
    └── ProductController.php
```

### Separate route files per version

```php
// routes/api.php — clean and organized
use Illuminate\Support\Facades\Route;

Route::prefix('v1')->name('v1.')->group(function () {
    require __DIR__ . '/api/v1.php';
});

Route::prefix('v2')->name('v2.')->group(function () {
    require __DIR__ . '/api/v2.php';
});
```

```php
// routes/api/v1.php
use App\Http\Controllers\Api\V1;

Route::apiResource('users',    V1\UserController::class);
Route::apiResource('posts',    V1\PostController::class);
Route::apiResource('products', V1\ProductController::class);
```

```php
// routes/api/v2.php
use App\Http\Controllers\Api\V2;

Route::apiResource('users',    V2\UserController::class);
Route::apiResource('posts',    V2\PostController::class);
Route::apiResource('products', V2\ProductController::class);
```

### Full api.php example

```php
<?php
// routes/api.php
use Illuminate\Support\Facades\Route;
use App\Http\Controllers\Api\V1;

// Public routes — no auth required
Route::post('/auth/login',    [V1\AuthController::class, 'login']);
Route::post('/auth/register', [V1\AuthController::class, 'register']);

// API v1 — protected routes
Route::prefix('v1')
    ->name('api.v1.')
    ->middleware(['auth:sanctum'])
    ->group(function () {

        // Auth
        Route::post('/auth/logout',  [V1\AuthController::class, 'logout']);
        Route::get('/auth/me',       [V1\AuthController::class, 'me']);

        // Resources
        Route::apiResource('users',    V1\UserController::class);
        Route::apiResource('posts',    V1\PostController::class);
        Route::apiResource('comments', V1\CommentController::class);

        // Nested resources
        Route::apiResource('posts.comments', V1\PostCommentController::class)
            ->only(['index', 'store']);

        // Admin only
        Route::middleware('role:admin')->group(function () {
            Route::apiResource('admin/users', V1\Admin\UserController::class);
        });
    });
```

### Generate resource controller

```bash
# Generate a controller with all resource methods
php artisan make:controller Api/V1/UserController --api

# Generate controller + model + migration in one command
php artisan make:model Post -mcr

# -m = migration
# -c = controller
# -r = resource controller (with all methods)
```

### Scoped route model binding

```php
// Ensure the comment belongs to the post
// Without scoping — anyone can access /posts/1/comments/99
// even if comment 99 belongs to post 2
Route::apiResource('posts.comments', CommentController::class)
    ->scoped(['comment' => 'slug']); // ensures comment belongs to post

// Or in route definition
Route::get('/posts/{post}/comments/{comment}', [CommentController::class, 'show'])
    ->scopeBindings();
```

## Best practices

- Always use `Route::apiResource()` for REST APIs — never `Route::resource()`
- Use route model binding — eliminates `find()` and manual 404 handling
- Organize controllers in `Api/V1/` folders from the start — easier to version later
- Use separate route files per version when api.php gets large
- Use scoped bindings for nested resources — prevents accessing child records from wrong parent
- Always protect API routes with `auth:sanctum` or similar middleware
- Use `->only()` or `->except()` to limit generated routes to what you actually need

<Callout type="tip">
  Generate your API controller with the right flags from the start:
```bash
  php artisan make:controller Api/V1/UserController --api --model=User
```
  This creates the controller with all 5 methods pre-filled and type-hinted with the `User` model — ready for route model binding immediately.
</Callout>

## Interview questions

<InterviewQ question="What is the difference between Route::resource and Route::apiResource?" difficulty="easy">
  `Route::resource()` generates 7 routes including `create` (GET form) and `edit` (GET form) routes that return HTML views. `Route::apiResource()` generates only 5 routes — index, store, show, update, destroy — skipping the HTML form routes since APIs return JSON not HTML forms. Always use `apiResource` for REST APIs.
</InterviewQ>

<InterviewQ question="How does route model binding work in Laravel?" difficulty="mid">
  When a route parameter name matches a controller method's type-hinted parameter name, Laravel automatically queries the database and injects the model instance. For example `{user}` with `User $user` automatically calls `User::find($id)`. If no record is found, Laravel returns a 404 response automatically. The binding key defaults to the primary key but can be customized with `getRouteKeyName()` or the `{user:slug}` syntax.
</InterviewQ>

<InterviewQ question="How do you implement API versioning in Laravel?" difficulty="mid">
  Group routes under a version prefix using `Route::prefix('v1')` and organize controllers in versioned namespaces like `App\Http\Controllers\Api\V1`. Old versions keep working while new versions introduce breaking changes. For large APIs, split each version into its own route file (`routes/api/v1.php`) and require it from `api.php`. This keeps the codebase organized and lets you deprecate old versions cleanly.
</InterviewQ>

<InterviewQ question="What is scoped route model binding and why is it important?" difficulty="hard">
  Scoped binding ensures a child model belongs to its parent model in nested routes. Without scoping, `GET /posts/1/comments/99` would return comment 99 even if it belongs to post 2 — a security issue. With `->scopeBindings()` or `->scoped()`, Laravel adds a `WHERE post_id = 1` constraint when fetching the comment, so it only resolves if the comment actually belongs to post 1. Returns 404 otherwise.
</InterviewQ>

## My notes

<MyNotes>
  Route model binding was the biggest productivity boost when I learned Laravel. I used to write `$user = User::findOrFail($id)` in every controller method. Now I just type-hint the model and Laravel handles everything.

  The custom key binding `{post:slug}` is incredibly useful for SEO-friendly URLs. Instead of `/posts/123` you get `/posts/my-awesome-post` and Laravel resolves the model by slug automatically.

  For API versioning — start with `V1` from day one even if you don't plan to have V2. It costs nothing and saves massive refactoring later when you inevitably need to make a breaking change.

  The artisan command that saves the most time:
```bash
  php artisan make:model Post -mcr --api
```
  Creates the model, migration, and resource controller all at once. I run this for every new resource.
</MyNotes>
